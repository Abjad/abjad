Creating Noteheads

The Abjad _NoteHead interface is a private class (as indicated by the _ before the class name). As such, it cannot be created directly and independently from a Note or Chord object. Note and Chord are the only two Abjad Components that have _NoteHead attributes. 

abjad> note = Note(0, (1, 4))
abjad> note.notehead
_NoteHead(c', 4)

abjad> chord = Chord([0, 2] (1, 4))
abjad> note.noteheads
[_NoteHead(c'), _NoteHead(d')]

Notice that a Note object has a _NoteHead interface directly attached to it, while the Chord object has a noteheads list of _NoteHead interfaces.

A _NoteHead interface inherits from the abstract _Interface, and thus carries all the _Interface attributes. See the _Interface reference for details.

Attribute examples

pitch

Return type

Returns an Abjad Pitch. 

abjad> note = Note(0, (1, 4))
abjad> note.notehead.pitch 
Pitch(c, 4)

Assignment Values

A Notehead pitch attribute can be set in two ways:

  • With a number representing the pitch in semitones from middle C.
  • With a Pitch object (Refer to the Pitch class section for details on Pitch initialization. 

abjad> note = Note(0, (1, 4))
abjad> note.notehead.pitch = 10
abjad> note
Note(bf', 4)

abjad> note.notehead.pitch = Pitch('cs', 5)
abjad> note
Note(cs'', 4)

style

Return type 

abjad> note = Note(0, (1, 4))
abjad> note.notehead.style == 'cross'
abjad> note.notehead.style 
'cross'

Assignment Values

A Notehead style must be set with a Python string naming the desired notehead style. 

abjad> note = Note(0, (1, 4))
abjad> note.notehead.style = 'cross'
abjad> f(note)
\once \override NoteHead #'style = #'cross
c'4
abjad> show(note)

Notice how the notehead is formatted when applying the formatting function f( ). Known or recognized styles, such as 'cross', will be formatted in the appropriate LilyPond input format. Unknown styles, such as 'myVeryOwnShape' is not an allowed LilyPond notehead style, and is thus formatted like so: 

abjad> note.notehead.style = 'myVeryOwnShape'
abjad> f(note)
\myVeryOwnShape
c'4

Unknown styles are permitted and formatted differently to allow the user to define his/her own styles with LilyPond identifiers. In this case, the user is responsible to define the identifiers for the show( ) function to properly compile the LilyPond output.

stylesSupported

Return type

The following are the supported styles:

  • 'cross'
  • 'parallelogram'
  • 'concavetriangle'
  • 'slash'
  • 'xcircle'
  • 'neomensural'
  • 'harmonic'
  • 'mensural'
  • 'petruccidiamond'
  • 'triangle'
  • 'semicircle'
  • 'diamond'
  • 'tiltedtriangle'
  • 'square'
  • 'wedge' 

abjad> note.notehead.stylesSupported
('cross', 'parallelogram', 'concavetriangle', 'slash', 'xcircle', 
'neomensural', 'harmonic', 'mensural', 'petruccidiamond', 'triangle', 
'semicircle', 'diamond', 'tiltedtriangle', 'square', 'wedge')
