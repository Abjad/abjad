The container duration interface implements a trivial read-only multiplier attribute, always equal to 1. I'm sure this is a way of creating an abstract attribute in the parent class that makes calculations on the tuplet duration interfaces easier. But this is a hack and should remove.

Look up _ContainerDurationInterface in containers/duration.py for implementation details.
