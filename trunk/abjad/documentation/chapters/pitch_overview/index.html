The Abjad Pitch class

The Abjad Pitch class models musical pitch.

Pitch conventions

Abjad follows these conventions regarding pitch:

  • middle C equals 0
  • numbers greater than 0 map to pitches higher than middle C
  • numbers less than 0 map to pitches lower than middle C
  • the octave of middle C equals octave number 4

Pitch initialization

Abjad provides four different ways to initialize Pitch objects.

  • integer
  • float
  • name, octave
  • pitch-class, number

Pitch initialization by integer

The usual way to instantiate Pitch objects is with integers. 

  abjad> Pitch(-10)
  Pitch(d, 3)

  abjad> Pitch(23)
  Pitch(b, 4)

Pitch initialization by float

For quartertones, you can initialize Pitch with floats. 

  abjad> Pitch(12.0)
  Pitch(c, 5)

  abjad> Pitch(12.5)
  Pitch(cqs, 5)

Pitch initialization by name, octave

Initalize Pitch with pitchclass and octave to specify accidentals explicitly. 

   abjad> Pitch('cs', 4)
   Pitch(cs, 4)

   abjad> Pitch('df', 4)
   Pitch(df, 4)

Pitch initialization by number, letter

Abjad will supply the correct accidental for a pitch of a given number if you supply the pitch letter against which to notate. 

   abjad> Pitch(13, 'c')
   Pitch(cs, 5)

   abjad> Pitch(13, 'd')
   Pitch(df, 5)

Pitch in the Abjad interpreter, pitch as LilyPond input

Abjad displays Pitch objects as (pitch-class, octave) pairs in the interpreter.

Abjad provides no format string for Pitch objects because Pitch objects should always be included as part of a notehead belonging to a note or chord for formatting.

Accidental spelling

Does pitch number 6 instantiate as F♯4 or as G♭4? Pitch-classes 0, 2, 4, 5, 7, 9 and 11 notate without accidentals. But a question remains as to what to do with the remaining pitch-classes. By default, Abjad maps pitch-classes to accidentals acording to the following table.

1 → C♯
3 → E♭
6 → F♯
8 → G♭
10 → B♭

Tonal music handles accidental spelling according to a system of keys. Abjad uses the accidental resolution method describe here as a type of neutral spelling for nontonal musics.

Comments

Do not confuse the object-oriented use of 'class' to describe the Abjad Pitch class with the use of 'class' in American pitch-class theory. The first refers to instantiable code, the second to the residue classes of a pitch space.

Abjad pitch specification with middle C equal to 0 follows the presentation of American pitch-class theory in, for example, Robert Morris's Composition with Pitch-classes (1978). IRCAM / MIDI pitch numbers equal Abjad pitch numbers plus 60.

Abjad allows integer and float pitch arguments only. Abjad does not allow pitch arguments of the form Rational(3, 2) or (3, 2). Use 1.5 instead.

Abjad currently generates a KeyError for pitch arguments of the form Rational(0, 2) and 0.2 but should generate a custom exception instead.

Abjad assigns sharps to pitch-classes 1 and 6 and flats to pitch-classes 3, 8 and 10. There is currently no way to change this behavior at note-instantiation time. But it is easy to change the accidental of any notehead after instantiation. To-do: design and implement some way to allow different acciental-assignment behaviors at note-instantiation time. The first strategy that comes to mind is the creation of a new Abjad cfg variable that can change during runtime — all sharps before instantiating an upward chromatic run, all flats before a downward chromatic run, etc.

Abjad implements quartertones but none of the other types of microtone. LilyPond now makes arrowed accidentals available and support for those is on the Abjad to-do list.

LilyPond implements several different accidental handling methods. Abjad outputs LilyPond input with no special indications for accidental handling. Nontonal music may prefer to use the LilyPond 'forget option for handling accidentals.

Do we have an Abjad-native way of setting 'forget?

Abjad currently allows the initialization of empty pitch with Pitch( ). Future versions of Abjad may deprecate this behavior.
